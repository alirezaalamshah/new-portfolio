اگر تازه با پایتون شروع کرده‌اید، احتمالاً با پیام‌های خطای نصب پکیج یا تداخل نسخه‌ها روبه‌رو شده‌اید. راه‌حل استاندارد این مشکل، **محیط مجازی** است.

## محیط مجازی چیست؟

محیط مجازی یک پوشه ایزوله است که نسخه پایتون و پکیج‌های هر پروژه را جدا از بقیه سیستم نگه می‌دارد. نتیجه: پروژه A می‌تواند Django 4 داشته باشد و پروژه B همزمان Django 5 — بدون هیچ تداخلی.

## ساخت محیط مجازی با venv

```bash
python -m venv .venv
```

فعال‌سازی در ویندوز:

```bash
.venv\Scripts\activate
```

و در لینوکس و مک:

```bash
source .venv/bin/activate
```

## نکته‌های مهم

- پوشه `.venv` را همیشه به `.gitignore` اضافه کنید.
- لیست پکیج‌ها را با `pip freeze > requirements.txt` ذخیره کنید.
- برای هر پروژه یک محیط جدید بسازید؛ محیط مشترک همان مشکل قبلی را برمی‌گرداند.

با همین سه دستور، دیگر هیچ‌وقت گرفتار تداخل پکیج‌ها نمی‌شوید.
